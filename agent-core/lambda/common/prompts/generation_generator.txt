Type mapping rules:
- char[N] → {"type": "string", "length": N}
- uint8_t / int8_t → "uint8" / "int8"
- uint16_t / int16_t → "uint16" / "int16"
- uint32_t / int32_t → "uint32" / "int32"
- uint64_t / int64_t → "uint64" / "int64"
- float → "float", double → "double"
- bool / uint8_t used as boolean → "uint8"
- StructType field[N] → {"type": "array", "length": N, "element": "StructType"}
  - If preceded by a count field (e.g., uint8_t count), use "count_field": "count" instead of "length"
- Nested struct (single) → use struct type name directly, define in "types"

Array rule: when a struct has a count field (e.g., uint8 count) immediately followed by an array, use "count_field" referencing that field, NOT a fixed "length".

Packet type values: convert hex string "0x0101" to decimal integer 257.

## phases section (REQUIRED)

Group C2S packets by gameplay phase based on the packet type category (high byte of type value).
Each phase maps to a list of category numbers.

Example:
"phases": {
  "Login": [1],
  "Enter Game": [2],
  "Gameplay": [4, 5, 6, 7],
  "Shop": [8],
  "Quest": [10],
  "Party": [11]
}

Category number = high byte of packet type (e.g., 0x0101 → category 1, 0x0801 → category 8).
Only include categories that have C2S packets.
