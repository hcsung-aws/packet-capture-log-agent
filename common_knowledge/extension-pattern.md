# 추가 기능 분리 패턴

## 규칙
기존 시스템에 기능을 추가할 때, 기존 코드를 수정하지 않고 새 파일 + 기존 인터페이스 구현으로 분리.

## 적용 조건
- 기존 시스템이 인터페이스/콜백으로 확장점을 제공할 때
- 추가 기능이 기존 기능의 "변형"이 아닌 "확장"일 때

## 구현 패턴
- 새 인터페이스 구현체 → 기존 시스템에 주입
- 기존 인터페이스 래핑 → 원래 구현체를 감싸서 기능 추가
- 진입점(main/CLI)에는 라우팅만 추가

## 안티패턴
- 기존 클래스에 모드 분기 추가 (if newMode then...)
- 기존 메서드 시그니처 변경
- 추가 기능을 위해 기존 테스트 수정

## 사례
패킷 캡처/재현 에이전트에 새로운 실행 모드를 추가할 때,
기존 캡처·재현 코드를 수정하지 않고:
- 새 파일에 모델 + 로직 구현
- 기존 인터셉터 인터페이스의 새 구현체로 동적 데이터 주입
- 기존 응답 핸들러 인터페이스를 래핑하여 상태 추적 추가
- 진입점에는 CLI 분기만 추가 (기존 메서드 불변)
결과: 기존 테스트 107개 영향 없이 신규 12개 추가, 총 119개 통과
